# Update Memory Template

Use this template after every lesson to keep the repository aligned with the learner's real progress.

## Update Order

1. Add new mistakes to `error-log.md` if they are recurring or important.
2. Add new words or phrases to `vocabulary-bank.md` if they are worth retaining.
3. Add grammar topics to `grammar-bank.md` only when the topic has actually appeared.
4. Add examiner-style observations to `examiner-notes.md` if they affect strategy.
5. Update `progress.md` and the relevant `sessions/` and `reports/` files.

## Memory Rules

- Avoid duplicate entries.
- Prefer stable phrasing that future LLMs can reuse.
- Keep examples short and directly tied to learner errors.
- If something is only useful for one day, keep it in the session file instead of long-term memory.

## Suggested Format

- Mistake: `incorrect form` → `correct form`
- Vocabulary: word, meaning, short example, status
- Grammar: topic, status, short example, common mistakes
- Examiner note: one concise observation with a strategic implication
